# 项目文件架构与 ISOM5240 提交映射

本文档依据课程文件 [ISOM5240_project_requirements.pdf](../ISOM5240_project_requirements.pdf) 中的 **Submission**、**Submission Checklist** 与 **Submission Guidelines** 整理：**本地开发仓库** 的推荐目录结构，以及如何打包为 **Canvas zip** 时的对应关系。

---

## 1. 与本项目相关的硬性要求（摘录）

- **至少两个 Hugging Face `pipeline`**：每个 pipeline 须使用 **Hugging Face 平台上的预训练模型**（不可用 OpenAI 等来源）。
- **微调与 App 一致**：Streamlit Cloud 上使用的模型须与 Notebook 中微调产出 **一致**，否则扣分。
- **Notebook**：助教将执行 **Restart & Run All**；输出应与报告一致；无法运行会扣分。
- **报告（PDF，&lt;10 页）**须含：题目与姓名、**公司名称与官网**、目标（≤50 词）、策略、**Model URL（HF 微调模型）**、**App URL（Streamlit Cloud）**、**GitHub URL**、数据集说明、模型与结构图、部署截图、实验（模型选型看 **准确率与运行时间**；Cloud 上应用性能以 **准确率** 为指标）、结论等。
- **实验结果**：另存 **Excel**（`Experimental_results.xlsx`），样本量需合理。
- **其他交付**：PPT、mp4（脸部可见）、数据集文件、**微调模型文件**、App 文件（`app.py`、`requirements.txt` 等）。

**与本应用设计的对应**：`text-generation`（预训练中文因果 LM，可选不微调）→ `text-classification`（在 Chinese-toxic 类数据上 **微调** 的检测模型）。课程若要求「每位学生一个主目标、只微调一个模型」，建议将 **微调对象定为分类模型**；生成侧使用第二个 HF pipeline 即可满足「双 pipeline」。

## 2. 本地仓库推荐目录结构（开发用）

在小组 GitHub 仓库根目录（与 Streamlit Cloud 绑定）建议采用下列布局，便于开发与对照 checklist：

```text
项目根目录/
├── docs/                              # 设计说明（辅助 Documentation，提交 zip 可选是否放入 documentation）
│   ├── 00_项目文件架构与提交映射.md
│   ├── 01_总功能流程.md
│   ├── 02_text_generation.md
│   ├── 03_文本敏感词与有毒内容检测.md
│   └── 04_分类模型微调与实验计划.md
├── notebooks/                         # 对应提交包中的 Python_notebooks（建议 Colab 同步或导出 .ipynb）
│   ├── 01_data_preprocessing.ipynb    # 数据加载、清洗、划分
│   ├── 02_model_finetuning.ipynb      # 分类模型微调、保存权重、（可选）上传 HF Hub
│   └── 03_experiments_and_evaluation.ipynb  # 对比实验、测试集评估、运行时间记录
├── app/                               # Streamlit 应用（与 GitHub_App_Files / Cloud 根目录一致）
│   ├── app.py
│   ├── requirements.txt
│   └── model_utils.py                 # 可选：懒加载 pipeline、路径常量
├── data/                              # 数据集（脱敏副本；体积大时用脚本说明下载方式）
│   ├── raw/                           # 原始或解压后数据
│   └── processed/                     # 清洗后、划分后的文件
├── models/
│   └── finetuned/                     # 提交用：App 加载的微调权重（与 Notebook 导出一致）
├── presentation/                      # 提交用
│   ├── Presentation_slide.pptx        # 按小组命名规范可改为课程要求文件名
│   └── grpXX.mp4                      # XX 为两位数组号，见 PDF
├── documentation/                     # 提交前导出（或从报告工作区复制进来）
│   ├── Project_report.pdf             # &lt;10 页
│   └── Experimental_results.xlsx
├── ISOM5240_project_requirements.pdf
├── 执行指南_代码生成规范.md
├── DL_NIJIALU_21237096.ipynb          # 个人参考/历史作业（若保留）
└── README.md                          # 建议写明：项目简介、本地运行步骤、三个 URL（HF / Streamlit / GitHub）
```

**说明**：

- 若 `data/` 或 `models/finetuned/` 过大，可在 `data/README.md` 中写明 **Hugging Face `datasets` 名称与版本** 及生成处理命令；zip 提交前按 TA 要求提供网盘链接。
- Streamlit Cloud 通常将 **仓库根目录** 或 **`app` 子目录** 设为入口；若 Cloud 指向子目录，请保证 `requirements.txt` 与 `app.py` 在同一路径层级，与课程「GitHub_App_Files」一致。

## 3. Canvas zip 打包映射（对照 PDF 示意图）

PDF 示例（组号与组长学号命名，如 `Group01_07123456.zip`）：

| zip 内路径（示例） | 本地仓库对应内容 |
|-------------------|------------------|
| `GroupXX_documentation/Project_report.pdf` | `documentation/Project_report.pdf` |
| `GroupXX_documentation/Experimental_results.xlsx` | `documentation/Experimental_results.xlsx` |
| `GroupXX_program/Python_notebooks/*.ipynb` | `notebooks/*.ipynb`（可保留编号前缀便于排序） |
| `GroupXX_program/GitHub_App_Files/` | `app/` 下全部文件（或 Cloud 实际部署的同一套文件） |
| `GroupXX_Dataset_files/` | `data/` 中需复现实验的文件（或说明 + 子集） |
| `GroupXX_Dataset_files/Fine-tuned_Model_files/` | `models/finetuned/`（与 App、Notebook 一致） |
| `GroupXX_presentation/*.pptx`、`grpXX.mp4` | `presentation/` |

另：**Streamlit Cloud App URL** 按作业要求单独在 Canvas 填写或附在报告首页。

## 4. 报告中的「模型结构图」建议（本课题）

与 PDF 示例 `image-to-text -> text generation -> text-to-audio` 类似，本项目可画为：

**用户输入（prompt）→ HF `text-generation`（预训练中文 LM）→ 生成文本 → HF `text-classification`（微调模型）→ 标签/分数 + 可选词典提示 → Streamlit 展示**

该图应写入 `Project_report.pdf` 的 Model 小节，并与 [01_总功能流程.md](01_总功能流程.md) 中的架构一致。

## 5. 提交前自检（Checklist）

- [ ] 报告含公司名、官网、三个 URL（HF 模型、Streamlit、GitHub）、数据集要点、模型图、部署截图、实验表、结论  
- [ ] Excel 记录全部实验，测试样本量合理  
- [ ] 所有 Notebook **Restart & Run All** 通过且输出已保存  
- [ ] `app.py` 使用的分类权重与 `notebooks/02_*` 导出及 `models/finetuned/` **一致**  
- [ ] 两个 pipeline 均为 HF 预训练模型；未使用 OpenAI 等禁用来源  
- [ ] zip 命名、视频命名符合 PDF「Submission Guidelines」  
- [ ] 大文件已按 TA 要求附下载链接（如需）
