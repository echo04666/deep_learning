# 分类模型微调计划与实验报告指引（Chinese-toxic）

本文档说明：**第二个模型**（文本分类 / 有毒内容检测）的数据集准备、微调流程、验证方法，以及如何根据实验结果撰写报告并与 **文本生成链路** 联动分析。

**已定方案**：**只微调单一基座模型** —— [google-bert/bert-base-chinese](https://huggingface.co/google-bert/bert-base-chinese)。**不更换** RoBERTa 等其他骨干。

**主训练数据（已定）**：[JunyuLu/ToxiCN](https://huggingface.co/datasets/JunyuLu/ToxiCN)（知乎、贴吧等中文社交平台语料，约 1.2 万条量级，带层次化标注）。加载后请以 `datasets` 实际 **`column_names`** 为准，将「正文」列统一映射为 `text`，**主标签**使用字段 **`toxic`（二分类）**。

**当前阶段对比实验（先做）** 在下列两维度展开：

1. **超参数**：学习率、`num_train_epochs`、`per_device_train_batch_size`、是否使用 **warmup**（及 `warmup_ratio` 或 `warmup_steps`）。
2. **数据与采样**：在 ToxiCN 上，训练集 **全量 vs 抽样子集**；是否对少数类做 **过采样 / 平衡采样**（与「不做平衡」对照）。

**后续阶段对比实验（暂缓）**：在 **相同模型与训练流程** 下，增加第二套训练数据 —— 基于 **常规敏感词词表**（如 [cjh0613/tencent-sensitive-words](https://github.com/cjh0613/tencent-sensitive-words)）**构造**的标注语料 —— 与 ToxiCN 训练得到的模型做效果对比（见 **§2.6**）。该部分可等主实验与报告主体完成后再做。

---

## 1. 任务与模型形式

- **任务（主实验）**：中文文本 **二分类** —— **有毒 / 无毒**，标签来自 ToxiCN 的 **`toxic`**（0=无毒，1=有毒，含义以数据集说明为准）。
- **基座模型（固定）**：`google-bert/bert-base-chinese` + `AutoModelForSequenceClassification`，使用 `Trainer` + `TrainingArguments` 微调。
- **交付物**：选定 **唯一最终配置**（在测试集上按主指标最优、且兼顾运行时间）后，将权重保存到 `models/finetuned/`（或上传至 Hugging Face Hub 并在报告中写 **Model URL**）；Streamlit App 必须加载 **与该次训练一致** 的 checkpoint（满足课程「App 与 Notebook 一致」）。

## 2. 数据集准备（主数据：JunyuLu/ToxiCN）

### 2.1 数据集来源与合规

- **主数据集**：`datasets.load_dataset("JunyuLu/ToxiCN")`（Hub 页面：[JunyuLu/ToxiCN](https://huggingface.co/datasets/JunyuLu/ToxiCN)）。
- **许可证**：数据集页面标注为 **CC BY-NC-ND 4.0**（非商业、禁止衍生再分发等）。请在报告与课堂说明中 **注明来源并引用论文**；若学校对数据集使用有额外要求，以教务/助教为准。
- **伦理**：官方含「样本可能令人不适」等提示；仅用于课程研究与检测实验，勿用于骚扰或自动化处罚他人等场景。
- **域说明**：语料来自 **中文社交平台**，与 **游戏 UGC** 存在 **域偏移**（黑话、缩写、语境不同），须在报告「局限」中说明。

### 2.2 ToxiCN 各标签字段含义（写给老师的说明）

ToxiCN 采用论文中的 **Monitor Toxic Frame** **层次化** 标注。除二分类外，还提供更细粒度字段（具体列名以 `load_dataset` 后 `column_names` 为准）。根据 Hub 数据集说明，各层含义概括如下：

| 字段（常见命名） | 取值含义（概括） | 说明 |
|------------------|------------------|------|
| **`toxic`** | 是否有毒：**0=无毒，1=有毒** | **最顶层、最直接对应产品需求**（「这条内容要不要拦」）。**本作业主实验只使用该字段做二分类。** |
| **`toxic_type`** | 有毒类型：**0=非毒，1=一般冒犯性言语，2=仇恨言论** | 在「已判定为有毒相关」下的粗分；与 `toxic` 有逻辑依赖，且 **3 类在「非毒」与两类有毒间分布需单独统计**，部分类别可能更稀疏。 |
| **`expression`** | 表达形式：**0=非仇恨语境，1=显性仇恨，2=隐性仇恨，3=举报/转述类** | 更细，涉及 **隐式有毒** 与 **引用他人脏话** 等，标注难度高；做多分类时样本不平衡与边界情况更多。 |
| **`target`（多为多标签/列表）** | 针对群体维度：如 LGBTQ、地域、性别歧视、种族歧视、其他、非仇恨 等（以数据集文档下标为准） | **多标签、多维度**，适合研究仇恨 **对象**，不适合作为课程首版 **单一多类头** 的快速落地；需多标签损失、采样策略等，超出当前「学生向、双 pipeline 应用」范围。 |

**为何主实验不用 `toxic_type` / `expression` / `target` 做训练目标（可向老师口头/报告陈述的要点）**：

1. **与系统目标一致**：Streamlit 流程里先做 **「有毒 vs 无毒」** 闸门，与 `toxic` 一一对应；细粒度标签可作为 **未来迭代**（先粗后细）。
2. **样本与不平衡**：细粒度类别 **样本更少、更不平衡**，同样规模下 **宏 F1 不稳定**，训练与解释成本高。
3. **层次依赖**：`toxic_type`、`expression` 等是在不同抽象层上的划分，简单当作互斥多类会 **忽略层次关系**，需更复杂建模（多任务/层次分类），不适合作为本课程初版唯一主线。
4. **作业边界**：课程重点在 **数据 → 微调 → 实验 → App**，二分类最易满足 **可复现、可写清、可与生成链路对接**。

若老师希望看到「多标签潜力」，可在报告 **Future Work** 中写：保留 ToxiCN 原始列，后续用 `toxic_type` 做多分类扩展实验。

### 2.3 预处理步骤

1. **加载**：`load_dataset("JunyuLu/ToxiCN")`，记录 revision 或快照日期（若需可复现）；打印 `column_names` 与首条样本，确认 **文本列名** 与 **`toxic`** 列。
2. **清洗**：
   - 去除空文本；
   - 过长样本 **截断** 到 `max_length`（与 tokenizer 一致）；
   - 标签缺失或异常样本剔除（用简单条件判断即可，逻辑写在 Notebook 中并注释）。
3. **划分**：
   - **训练集 / 验证集 / 测试集**（例如 80% / 10% / 10%，或按数据集已有 split 使用）；
   - **分层抽样**划分（若标签不平衡，尽量保持各 split 中类别比例接近全量分布）。
4. **统计**：用表格记录各 split 样本数、**`toxic` 正负例占比**，复制到实验报告或 Excel。

### 2.4 实验用「数据条件」（与第 3、6 节配合）

在 **同一套 train/val/test 划分** 下，仅改变 **用于训练的数据构造方式**（验证集、测试集保持不变，保证可比）：

| 数据条件 ID | 训练集构造 | 说明 |
|-------------|------------|------|
| **D1_full** | 训练集 **全量** | 不做子集抽样。 |
| **D2_sample** | 训练集 **随机抽样子集** | 例如固定 `SEED` 后取 30% / 50% 样本（比例在 Notebook 常量中写死并报告）；**分层抽样**以保持类别比。 |
| **D3_balance_off** | 与 D1 或 D2 二选一组合 | **不做** 过采样；按自然分布训练。 |
| **D4_balance_on** | 在与上相同的规模下 | 对 **少数类过采样**（或有放回采样使每 epoch 内近似平衡）或等价简单策略；实现时用 **显式循环 / pandas 采样** 即可，避免复杂框架。 |

**报告写法**：先说明全量 vs 抽样的动机（速度、模拟小数据场景）；再说明平衡 vs 不平衡对 **宏 F1 / 少数类召回** 的预期影响。

### 2.5 与游戏 UGC 的关系（报告必写）

- 训练数据通常 **不是** 游戏内聊天语料，因此分类器在游戏黑话、缩写上的性能可能下降。
- **建议实验**：用 [02_text_generation.md](02_text_generation.md) 中的生成模块采样一批文本（含中性 prompt 与边界 prompt），用 **最终选定的微调 checkpoint** 打分，在报告中做案例分析（见第 5 节）。

### 2.6 后续（阶段二、暂缓）：敏感词词库构造数据集 vs ToxiCN

**动机**：ToxiCN 来自 **真实用户评论**，反映「社交平台上的有毒表达」；**敏感词表**（如腾讯词库）反映的是 **运营合规清单**，二者 **分布不同**。你希望做 **两种训练数据对比**，在报告里讨论「真实标注语料 vs 词表归纳信号」的差异 —— **合理且有价值**。本阶段 **不阻塞** 当前主实验；待 ToxiCN 主线的超参 + 抽样/平衡实验与 App 打通后再做。

**词库数据为何没有「天然」句子标签**：词表通常是 **词条列表**，不含 `(句子, 0/1)`。要做监督学习，需要 **自行构造** 训练集，例如（实现时选一种简单、可写进报告的方法即可）：

- **正例（label=1）**：包含词表中词/短语的 **合成句或模板句**、或从爬取语料中 **命中词表** 的句子（注意隐私与版权）。
- **负例（label=0）**：来自 **新闻/维基摘要/开源中文平行语料** 等相对「干净」的句子，且经简单规则过滤长度与空字符。

**必须在报告中写清的局限**（避免被老师质疑方法论）：

- 词库数据往往是 **弱监督 / 启发式标注**，**「命中敏感词 ≠ 句子一定有毒语境」**（否定、引用、反讽）；负例也可能与游戏场景差异大。
- 与 ToxiCN **领域不同**，直接比较训练 loss 意义有限；更合理的对比是：**在同一固定测试集上评指标** —— 建议 **统一在 ToxiCN 的 test split（或合并后的 hold-out）上报告 Accuracy / 宏 F1**，看「用 ToxiCN 训练」vs「用词库构造集训练」谁 **泛化到同一批真实标注数据** 更好；若时间允许，可再补 **小批生成文本** 上的定性对比。

**实施顺序建议**：先完成 §3–§6 的 ToxiCN 主实验与最终 checkpoint；再在 `notebooks/` 中新增「词库数据构造 + 训练 + 同测试集评估」小节，结果填入 **§6.4**。

## 3. 超参数与训练设置（单一模型上的网格）

**固定**：`MODEL_NAME = "google-bert/bert-base-chinese"`，`MAX_LENGTH`（如 128 或 256）、`SEED`、`weight_decay`（若不变则写进报告常量）。

**变化因子（实验 A 组：超参数）** —— 在 **ToxiCN**、**同一数据条件**（建议先用 **D1_full + D3_balance_off**）下扫描，记录 **验证集** 主指标与 **每 epoch  wall time** 或总训练时间（满足课程「模型选型：准确率与运行时间」）。

| 因子 | 建议取值（示例，可在 Notebook 中改） |
|------|--------------------------------------|
| 学习率 `learning_rate` | 如 `2e-5`, `3e-5`, `5e-5` |
| `num_train_epochs` | 如 `2`, `3`, `4` |
| `per_device_train_batch_size` | 如 `8`, `16`（OOM 则减小） |
| Warmup | **关**：`warmup_steps=0` 且不设 ratio；**开**：`warmup_ratio=0.1`（或固定 `warmup_steps`，二选一写清） |

**做法建议**：

- 不必做完整笛卡尔积（组合爆炸）；选 **6～12 组** 有代表性的组合即可（例如固定 epoch=3，扫学习率 + batch；再固定最优学习率，扫 epoch 与 warmup 开/关）。
- 每组在 **同一验证集** 上比较；**最终测试集** 只在选定最佳配置后评估 **一次**，避免反复调测试集。

**学生向实现**：超参放在 Notebook **设置单元格** 的字典或常量区；用简单 for 循环依次 `Trainer` 训练并 `trainer.evaluate()`，结果 append 到 list 再 `pandas.DataFrame` 导出（便于复制到 Excel）。

## 4. 验证方法

### 4.1 主指标（测试集上报告）

- **准确率** Accuracy；
- **精确率、召回率、F1**（标签不平衡时重点报告 **宏平均 F1** 与 **微平均 F1**）；
- **混淆矩阵**；
- 若为多分类：可附 **逐类 F1** 表。

### 4.2 验证流程

1. **超参实验（A 组）**：用验证集比较各组；记录 **验证集指标 + 训练总时长**（或单 epoch 时间）。
2. **数据实验（B 组）**：在 **已选定的较优超参** 下，切换 **D1/D2** 与 **balance on/off**，同样先看验证集。
3. **测试集**：确定 **唯一最终配置**（数据条件 + 超参）后，在 **测试集上只评一次**，写入报告主表。
4. 在 Notebook 中保存 **预测样例**（true positive / false positive / false negative）供报告「错误分析」引用。

### 4.3 可选扩展（时间允许）

- 小样本人工抽检；简单改写样本上的分数变化（探索性）。

2. 用生成模块得到文本样本集。
